# 💳 Система ручной оплаты MasterConnect

## 📋 Обзор

Система ручной оплаты позволяет студентам загружать доказательства оплаты (скриншоты/PDF чеков), а администраторам - проверять и подтверждать оплату вручную.

## 🔄 Процесс оплаты

### 1️⃣ Студент создает бронирование
- Выбирает ментора и время консультации
- Заполняет форму заявки
- Бронирование создается со статусом **HOLD** (10 минут на оплату)

### 2️⃣ Студент загружает доказательство оплаты
- Нажимает кнопку **"Оплатить"** на странице бронирований
- Загружает скриншот или PDF чек об оплате
- Может добавить комментарий к оплате
- Статус меняется на **AWAITING_VERIFICATION** (ожидает проверки)

### 3️⃣ Администратор проверяет оплату
- Видит бронирования со статусом **AWAITING_VERIFICATION** в админ-панели
- Нажимает кнопку **"Проверить"** или **"Проверить оплату"** в меню действий
- Просматривает загруженные доказательства оплаты
- Может добавить заметки и номер транзакции
- **Подтверждает** или **Отклоняет** оплату

### 4️⃣ Результат проверки
- **При подтверждении**: статус → **CONFIRMED** (подтверждено)
- **При отклонении**: статус → **REJECTED** (отклонено)

## 🎨 Интерфейс

### Для студентов

#### Страница "Мои сессии" (`/student/bookings`)
- Кнопка **"Оплатить"** для бронирований со статусом HOLD
- Индикатор **"Ожидает проверки"** для AWAITING_VERIFICATION

#### Дашборд студента (`/student/dashboard`)
- Быстрый доступ к оплате предстоящих бронирований
- Отображение статуса проверки оплаты

#### Модальное окно загрузки
- Drag & Drop загрузка файлов
- Поддержка форматов: **PDF, JPG, PNG**
- Максимальный размер: **10MB**
- Поле для комментария (необязательно)
- Валидация на клиенте и сервере

### Для администраторов

#### Страница "Бронирования" (`/admin/bookings`)
- Фильтр по статусу **AWAITING_VERIFICATION**
- Быстрая кнопка **"Проверить"** в таблице
- Меню действий с опцией **"Проверить оплату"**

#### Модальное окно проверки
- Список всех загруженных доказательств
- Предпросмотр изображений
- Кнопка открытия PDF в новой вкладке
- Поля для заметок и номера транзакции
- Кнопки **"Подтвердить"** и **"Отклонить"**

## 🔧 Технические детали

### Backend API

#### Endpoints для студентов
```
POST /api/v1/payments/evidence
- Загрузка доказательства оплаты
- Параметры: booking_id, comment_text, file (multipart/form-data)
- Возвращает: PaymentEvidence

GET /api/v1/payments/evidence/booking/{booking_id}
- Получение всех доказательств для бронирования
- Возвращает: List[PaymentEvidence]
```

#### Endpoints для администраторов
```
POST /api/v1/bookings/{booking_id}/admin/confirm-payment
- Подтверждение или отклонение оплаты
- Параметры: payment_confirmed, payment_notes, payment_reference
- Возвращает: Booking
```

### Frontend компоненты

#### `PaymentEvidenceUpload.tsx`
- Компонент для загрузки доказательств оплаты студентами
- Drag & Drop, валидация, превью файлов

#### `PaymentEvidenceViewer.tsx`
- Компонент для проверки доказательств администраторами
- Просмотр файлов, добавление заметок, подтверждение/отклонение

## 🔐 Безопасность

### Валидация файлов
- Проверка типа файла (MIME type)
- Проверка расширения (.pdf, .jpg, .jpeg, .png)
- Ограничение размера (10MB)
- Валидация на клиенте и сервере

### Права доступа
- Студенты могут загружать доказательства только для своих бронирований
- Администраторы могут просматривать все доказательства
- Менторы могут просматривать доказательства для своих бронирований

### Аудит
- Логирование всех действий с доказательствами
- Сохранение информации о создателе
- Отслеживание изменений статуса бронирования

## 📊 Статусы бронирования

| Статус | Описание | Действия студента | Действия админа |
|--------|----------|-------------------|-----------------|
| **HOLD** | Временная бронь (10 мин) | Загрузить доказательство | - |
| **AWAITING_VERIFICATION** | Ожидает проверки | Ожидание | Проверить оплату |
| **CONFIRMED** | Оплата подтверждена | - | Создать Zoom, Отменить |
| **REJECTED** | Оплата отклонена | - | - |
| **COMPLETED** | Консультация завершена | Оставить отзыв | - |
| **CANCELLED** | Бронирование отменено | - | - |

## 📝 Примечания

### Хранение файлов
✅ **Файлы сохраняются на сервере** в папку `uploads/payment_evidences/`

**Текущая реализация:**
- Файлы загружаются в локальную папку `uploads/payment_evidences/`
- Генерируется уникальное имя файла: `{uuid}.{extension}`
- URL файла: `/uploads/payment_evidences/{filename}`
- Файлы раздаются через FastAPI StaticFiles

**Для production (опционально):**
Если нужно использовать S3-совместимое хранилище:
1. Настроить S3 (AWS S3, MinIO, DigitalOcean Spaces)
2. Обновить `apps/api/src/modules/payments/api/routes.py`:
   - Заменить сохранение на диск на загрузку в S3
   - Использовать `boto3` или аналогичную библиотеку
3. Настроить переменные окружения в `.env`:
   ```
   S3_ENDPOINT=https://s3.amazonaws.com
   S3_BUCKET=masterconnect-payments
   S3_ACCESS_KEY=your_access_key
   S3_SECRET_KEY=your_secret_key
   S3_REGION=us-east-1
   ```

**Преимущества текущей реализации:**
- ✅ Простота настройки
- ✅ Нет зависимости от внешних сервисов
- ✅ Быстрая загрузка и доступ к файлам
- ✅ Подходит для малых и средних проектов

**Когда нужен S3:**
- Большой объем файлов (>100GB)
- Распределенная инфраструктура
- CDN для быстрой доставки по всему миру
- Автоматическое резервное копирование

### Безопасность URL
Рекомендуется использовать:
- Подписанные URL (presigned URLs) с ограниченным временем жизни
- CDN для быстрой доставки файлов
- Watermarking для защиты от копирования

## 🐛 Известные проблемы

1. ~~**Файлы не загружаются на реальный сервер**~~ - ✅ **ИСПРАВЛЕНО**: Файлы сохраняются в папку `uploads/payment_evidences/`

3. **Нет автоматического удаления старых файлов** - требуется cron job для очистки

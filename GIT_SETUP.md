# Инструкция по загрузке проекта на GitHub

## Шаг 1: Проверка .gitignore

Убедитесь, что `.gitignore` правильно настроен и исключает:
- `.env` файлы (кроме примеров)
- `*.db` файлы (SQLite базы данных)
- `credentials/` папку
- `uploads/` папку
- `venv/` и `node_modules/`
- Все чувствительные данные

## Шаг 2: Добавление файлов в Git

```bash
# Проверить какие файлы будут добавлены
git status

# Добавить все файлы (кроме исключенных в .gitignore)
git add .

# Проверить что добавлено
git status
```

## Шаг 3: Первый коммит

```bash
git commit -m "Initial commit: MasterConnect platform with Docker production setup"
```

## Шаг 4: Загрузка на GitHub

```bash
# Установить основную ветку (если нужно)
git branch -M main

# Загрузить на GitHub
git push -u origin main
```

## Важно перед пушем:

1. **Проверьте что НЕ попадет в репозиторий:**
   ```bash
   git status
   ```
   Убедитесь что:
   - `.env` файлы НЕ добавлены
   - `test.db` и другие базы данных НЕ добавлены
   - `credentials/` папка НЕ добавлена
   - `uploads/` папка НЕ добавлена

2. **Если что-то чувствительное попало:**
   ```bash
   # Удалить из индекса (но оставить файл локально)
   git rm --cached .env
   git rm --cached test.db
   # и т.д.
   ```

3. **Проверить размер репозитория:**
   - `venv/` и `node_modules/` должны быть исключены
   - Если они большие, проверьте `.gitignore`

## После успешного пуша:

1. На GitHub проверьте что все файлы загружены
2. Убедитесь что `.env` файлы НЕ видны на GitHub
3. Проверьте что `credentials/` папка НЕ видна
4. Убедитесь что примеры конфигурации (`.env.example`, `.env.production.example`) видны

## Если нужно обновить репозиторий:

```bash
git add .
git commit -m "Описание изменений"
git push
```

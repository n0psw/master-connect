## Найденные проблемы и рекомендации

### 🔴 Критические проблемы

**Нет критических проблем**

### ⚠️ Важные проблемы (ИСПРАВЛЕНО)

1. **Неправильный endpoint в frontend API клиенте** ✅ ИСПРАВЛЕНО
   - **Файл:** `apps/web/src/shared/api/bookings.ts:92`
   - **Проблема:** Метод `rejectPayment` использует несуществующий endpoint
   - **Решение:** Обновлен метод для использования `confirm-payment` с `payment_confirmed: false`

2. **Синтаксис валидатора в Pydantic схеме** ✅ ИСПРАВЛЕНО
   - **Файл:** `apps/api/src/modules/bookings/domain/schemas.py:27`
   - **Проблема:** Использовался старый синтаксис `@validator` вместо `@field_validator` для Pydantic v2
   - **Решение:** Обновлен валидатор `validate_questions` на `@field_validator` с `@classmethod` декоратором
   - **Примечание:** Остальные валидаторы в файле используют старый синтаксис, но Pydantic v2 поддерживает обратную совместимость, поэтому они работают корректно

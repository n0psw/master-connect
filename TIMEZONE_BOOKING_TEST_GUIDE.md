## Автоматизированный Тест (Python)

```python
import pytest
from datetime import datetime, timezone
import pytz

def test_slot_generation_cross_timezone():
    """Тест генерации слотов для разных timezone"""
    
    # Ментор в Москве (UTC+3)
    mentor_tz = pytz.timezone("Europe/Moscow")
    mentor_time = datetime(2025, 1, 10, 9, 0)  # 09:00 местное время
    mentor_time_aware = mentor_tz.localize(mentor_time)
    mentor_time_utc = mentor_time_aware.astimezone(pytz.UTC)
    
    # Проверяем что 09:00 Москва = 06:00 UTC
    assert mentor_time_utc.hour == 6
    
    # Студент в Алматы (UTC+5)
    student_tz = pytz.timezone("Asia/Almaty")
    student_view = mentor_time_utc.astimezone(student_tz)
    
    # Проверяем что студент видит 11:00 (06:00 UTC + 5 часов)
    assert student_view.hour == 11
    
    print(f"✅ Ментор работает: {mentor_time_aware.strftime('%H:%M')} (Москва)")
    print(f"✅ Сохранено в БД: {mentor_time_utc.strftime('%H:%M')} UTC")
    print(f"✅ Студент видит: {student_view.strftime('%H:%M')} (Алматы)")

if __name__ == "__main__":
    test_slot_generation_cross_timezone()
```

## Известные Ограничения

1. **Browser Timezone Detection**: Фронтенд использует `Intl.DateTimeFormat().resolvedOptions().timeZone` для определения timezone браузера. Если пользователь не установил timezone в профиле, используется timezone браузера.

2. **Daylight Saving Time (DST)**: pytz и dayjs автоматически учитывают переход на летнее/зимнее время.

3. **Cache**: Календарь доступности кэшируется на 60 секунд. При изменении timezone ментора может потребоваться до 60 секунд для обновления.

## Что Делать Если Тест Провалился

1. **Проверьте timezone ментора**:
   ```sql
   SELECT timezone FROM mentor_settings WHERE mentor_id = '...';
   ```

2. **Проверьте timezone студента**:
   - Откройте Dev Tools → Application → Local Storage
   - Найдите user data и проверьте поле `timezone`

3. **Проверьте API response**:
   - Network Tab → найдите запрос к `/availability/mentors/{id}/calendar`
   - Проверьте поле `timezone` в response (должен быть timezone ментора)
   - Проверьте `slots[0].start` (должен быть UTC timestamp)

4. **Очистите кэш**:
   - Backend: перезапустите сервер (очистит in-memory cache)
   - Frontend: Ctrl+F5 или очистите кэш браузера

# Ограничение публичной регистрации только для студентов

## Проблема

Сейчас в форме регистрации можно выбрать роль (Студент/Ментор). Нужно:

- Публичная регистрация только для студентов
- Менторы регистрируются через администратора

## Решение 1: Упростить форму регистрации

### 1.1 Убрать выбор роли из RegisterPage

- Удалить блок с выбором роли (радио-кнопки Студент/Ментор)
- Автоматически устанавливать роль "student" при регистрации
- Обновить validation schema (убрать поле role)

### 1.2 Упростить UI регистрации

- Удалить иконки и выбор роли
- Упростить дизайн формы
- Обновить тексты и описания

## Решение 2: Проверить функционал создания менторов в админке

### 2.1 Изучить существующий функционал админки

- Проверить `apps/web/src/pages/admin/AdminUsersPage.tsx`
- Проверить `apps/web/src/shared/api/admin.ts`
- Найти функционал создания пользователей админом

### 2.2 Проверить backend endpoints

- Изучить `apps/api/src/modules/admin/api/routes.py`
- Проверить есть ли endpoint для создания пользователей
- Проверить endpoint для создания менторов

### 2.3 Реализовать функционал если его нет

- Создать endpoint для создания ментора администратором (backend)
- Создать форму/модальное окно для добавления ментора (frontend)
- Реализовать полный flow: создание user + mentor profile

## Решение 3: Тестирование

### 3.1 Протестировать регистрацию студентов

- Убедиться что регистрация работает без выбора роли
- Проверить что создается профиль студента

### 3.2 Протестировать создание менторов через админку

- Войти как администратор
- Создать нового ментора через админ-панель
- Убедиться что создается user + mentor profile
- Проверить что ментор может войти в систему
